Algorithms

In computer science an algorithm is a set of coding instructions that is used to solve a specific problem. On the algorithms contained here are for solving the issue of searching and sorting sets of data. Whether the data is as simple as numbers or something more advanced i.e sorting objects via an id value the approach is more or less the same. In each of the specific algorithms we will show how it works and how to implement it. (In this area the only algorithm that is fleshed out is the merge sort algorithm. All others have a page but contain blank data and greeked layout text.)

Merge sort

Merge sort is a highly efficient and useful comparison based algortihm which can be used in a multitude of situations to sort data. Using the divide and conquer based strategy this algorithm is widely used. Merge sort is a very consistant algorithm which makes it an attractive one to use. Merge sort

Above we see two demonstrations of how merge sort works, by splitting each part in half and consistantly doing this then reforming as shown above. We see divide and conquer working, the space complexity is order(n) due to the fact we have to dismantle and create new mini arrays when splitting it up. The best, average and worst case time complexity for merge sort is always (n log(n)) which along side heapsort makes it one of the most consistant algorithms for time to completion which often makes merge sort desireable.
Underneath we see one version of code implementation which is shown in java. The below code will take a small array, divide and counquer and sort the array back together again, it is a very small and simple example, however shows exactly how it works. The below code can also be modified to take input of arrays, or take an array of any given size with any given type, and sort it.



Regarding code implementation

Ok lets break the code down and explain exactly how it works. If we move to the main function first at line 80 onward, we see we have an array of 8 elements, the included numbers are in-material, then we create a new mergesort object. First we print the result fo the array as is before applying merge sort, which simply outputs the array. Next we enter the array into the merge_sort function, it takes the array, start element 0, and end element 7. Next onto the merge_sort function, it checks if l is less than r which at this point of course it will be i.e. 0 is less than 7. Next it gets the mid point of the array, and we add the left half and the right half back into merge_sort again. This continues until the left element is not less than the right i.e. there is only one element.


Finally we begin merging the two halves as demonstrated in the image and video at the top of this page. We compare the internal numbers and moving their values consistantly until the whole array is reconstructed in the correct order. Finally the array is again printed after merging and we get a nicely sorting array in for this case an almost instant result. Obviously due to the time complexity of merge sort the larger this array becomes the longer it will take. To view more on complexity view the complexity page.
